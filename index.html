What It Is:

Cryptography is about constructing and analyzing protocols that prevent third parties or the public from reading private messages.

How it Works:

Cryptography relies on the use of encyption to scramble the contents of a message where they can later be un-scrambled and processed.

Who Uses It:

Every Computer when trying to communicate privately. This could be passwords, emails, purchasing information, billing adresses.

Are there Different types?

Symmetric Key Cryptography (SKC): Uses a single key for both encryption and decryption
Asymmetric Key Cryptography (AKC): Uses one key for encryption and another for decryption (They are mathematically related; however, one cannot infer one key, specifically the private key knowing the other, the public key.)

Hash Functions: Uses a one-way mathematical transformation that is hard to reverse.

Symmetric Key Cryptography(SKC):

SKC is essentially the concept of Ceasars cypher which is sending a message that that the recipiant already has an identicle key for. For Example: if the key instructs the recipient to count 5 letters further for each letter in order to discramble the message. The problem with this is it is fairly easy to crack. There are only 26 combinations that these letters in this order could be. To solve this, keys instruct a different amount of numbers to rotate for each character. This would take lifetimes for a person to crack as billions of combinations must now be tried. However, computers can still complete these combinations within minutes. This is why computers use 264-Bit keys that even an army of computers can not crack in centuries. It is important to understand that technology has not made these keys more difficult to crack but instead exponentially increases the amount of guesses required to crack these codes. In the years to come, as the processors of computers get more and more advanced, these 264-Bit keys may not hold up. But that is just for regular computers, Quantum Computers already pose a massive threat.

Asymmetric Key Cryptography(AKC)

Asymmetric key cryptography, also known as public-key cryptography, uses two different keys (public and private). The public key is for the sender who encrypts data while the recipeint uses the private key to decrypt messages. Since the public key is public, The owner(s) of the private key are the only ones who can decrypt the messages. This makes it possible to securely exchange messages without a prior shared secret.
